# Mobile App Design

## Purpose

The mobile app must enforce good capture. It should not behave like a generic image uploader.

---

## Required Screens

- crop selection
- sample preparation guide
- calibration grid placement guide
- top-view capture
- angled-view capture
- flash/no-flash paired capture
- image-quality feedback
- results screen
- retake screen
- history/report screen

---

## Camera Controls

- exposure lock
- white balance lock
- focus lock
- zoom lock
- distance guide
- framing overlay
- flash/no-flash pairing

---

## Result Screen

Should show:

- moisture risk
- confidence
- quality grade
- defect percentages
- estimated moisture percent if calibrated
- recommended next action

---

## UX Rule

If the app sees a screenshot or invalid image, it must block submission and ask the user to capture a real sample image.
